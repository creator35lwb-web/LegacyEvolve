# Contributing to LegacyEvolve Protocol

Thank you for your interest in contributing to the LegacyEvolve Protocol (LEP)! We welcome contributions from the community to help shape the future of AI-legacy integration.

## How to Contribute

### 1. Protocol Design Feedback

As we are currently in the protocol design phase, the most valuable contributions are feedback and suggestions on the protocol specification. You can:

-   Open an issue to discuss a specific aspect of the protocol.
-   Submit a pull request with proposed changes to the `docs/ARCHITECTURE.md` file.

### 2. Adapter Development

Once the Adapter Development Kit (ADK) is released, you can contribute by:

-   Building adapters for new legacy systems.
-   Improving existing adapters.
-   Contributing to the ADK itself.

### 3. Documentation

Help us improve our documentation by:

-   Fixing typos and grammatical errors.
-   Adding examples and tutorials.
-   Translating documentation into other languages.

## Code of Conduct

Please be respectful and constructive in all interactions. We are committed to providing a welcoming and inclusive environment for all contributors.

## Getting Started

1.  Fork the repository.
2.  Create a new branch for your changes.
3.  Make your changes and commit them with clear, descriptive messages.
4.  Submit a pull request.

We look forward to your contributions!
